Текстовий матеріал

Команди


Програма — це набір (список) команд. Спочатку виконується перша команда, потім друга, третя й так далі. Після виконання всіх команд програма завершується.


Програми, написані мовою Java, виконує спеціальна програма JVM (Java Virtual Machine — віртуальна машина Java), яка вміє їх виконувати.


Але ми почнемо не з команд, а з декількох простих принципів.


Принцип перший: у мові програмування Java кожна команда має починатися з нового рядка і закінчуватися крапкою з комою.


Приміром, ми хочемо двічі вивести на екран напис Я люблю JAVA!. Ось як виглядатиме код цієї програми:


System.out.println("Я люблю JAVA!");
                        System.out.println("Я люблю JAVA!");

Принцип другий: програма не може містити просто список команд. У мові Java команди мають міститися у функціях, а функції — у класах.


Java-програми складаються з класів, класи містять методи, а методи — команди.



Структура типової програми


Програми мовою Java складаються з класів. Класів може бути десятки тисяч. Мінімальна програма — це один клас. Для кожного класу створюється окремий файл, ім'я якого збігається з ім'ям класу.


Наприклад ви вирішили створити клас, який описує будинок. Тоді вам потрібно створити клас House, який зберігатиметься у файлі House.java.


Ці файли містять текст — код мовою програмування Java. Зазвичай код класу складається з імені класу й тіла класу. Тіло класу беруть у фігурні дужки. Наприклад:


public class java_course {
                            //Тут знаходиться тіло класу
                        }

Тіло класу може містити змінні (інакше — дані класу) і методи (функції класу).


public class java_course {
                            int a; //змінна
                            void funk() {} //метод
                        }

У наведеному вище прикладі a — це змінна, а funk — метод.



Метод main()


Класи можуть містити змінні й методи, але це не обов'язково. Трапляються класи без змінних чи без методів.


До складу мінімальної програми має входити принаймні один клас, що має містити принаймні один метод (функцію), з якого починається виконання програми. Такий метод повинен мати ім'я main. Мінімальна програма має такий вигляд:


public class java_course {
                            public static void main(String[] args) {}
                        }

Зауважте, що в наведеному вище прикладі метод main не містить команд. Саме так: у мінімальній програмі немає жодної команди. На те вона й мінімальна.


Клас, з якого починається програма, може мати будь-яке ім'я, однак вигляд методу main, з якого розпочинається виконання програми, завжди буде однаковим:


public class java_course {
                            public static void main(String[] args) {
                                // Код методу
                            }
                        }


Виведення на екран


Параметри методу println()


Тіло методу складається з команд. Можна навіть сказати, що метод — це група команд, якій присвоєно ім'я (ім'я методу). І так, і так буде правильно.


Команди можуть бути різними. У мові Java є команди на всяку потребу. Кожна команда описує певну дію. Кожна команда має закінчуватися крапкою з комою.


public static void main(String[] args) {
                            // Виводить на екран число 336
                            System.out.println(336);
                            // Виводить на екран напис: Java course
                            System.out.println("Java course");
                        }

Насправді це одна команда — System.out.println. А в круглих дужках вказано параметри, які їй передаються. Одна команда може виконувати різні дії залежно від значень параметрів. Це дуже зручно.


Важливо!
У мові Java має значення, якими літерами написано команди — великими чи малими. Команда System.out.println() буде виконуватись, а команда system.out.println() — ні.


Якщо ви хочете вивести на екран текст, його потрібно з обох сторін позначити подвійними лапками.


Одинарна лапка (апостроф) має такий вигляд: ', а подвійна лапка — такий: ". Не слід плутати подвійні лапки з парою одинарних — це зовсім різні знаки.


Різниця між командами println() і print()


Команда виведення на екран має два варіанти: System.out.println() і System.out.print()


Якщо написати команду System.out.println() декілька разів, то переданий в неї текст буде щоразу виводитися з нового рядка. А якщо написати команду System.out.print(), текст виводитиметься в тому самому рядку.


System.out.println("Java");
                        System.out.println("course");

Результат:


Java
course


System.out.print("Java");
                        System.out.print("course");

Результат:


Java course



Змінні


Визначення


Змінні — це такі штуковини, де зберігаються дані. Будь-які дані. У Java всі дані зберігаються у змінних. За своїм призначенням змінна дуже схожа на коробку.


Кожна змінна в Java має три важливі властивості: тип, ім'я та значення


Ім'я потрібно для того, щоб відрізнити одну змінну від іншої. Так само, як і напис на коробці.


Тип змінної вказує на тип значень чи даних, які можна в ній зберігати. У коробці для цукерок ми зберігаємо цукерки, в коробці для взуття — черевики і т. ін.


Значення — це певний об'єкт, дані чи інформація, що зберігаються у змінній.


У мові Java кожен об'єкт має свій тип. Наприклад, є такі типи даних: ціле число, дробове число, текст тощо.


Кожна змінна (коробка) також має свій тип. У змінній можна зберігати тільки такі значення, тип яких відповідає типу змінної. Різні коробки використовують для зберігання різних речей: коробка для цукерок, коробка для яєць тощо. Усе як в житті.


Створення змінної


У мові Java для створення змінної використовується така команда: тип ім'я; де тип — це тип змінної (збігається з типом значень, які можуть зберігатися у змінній), а ім'я — це ім'я змінної. 


int a; // Створюємо змінну з іменем a типу int.
                        String s; // Створюємо змінну з іменем s типу String.

Найчастіше використовують два типи — int (цілі числа) і String (текст). Крім того, часто використовують тип double — це дробові (дійсні) числа.


Присвоювання


Для запису значень у змінну є спеціальна операція — операція присвоювання. Вона копіює значення з однієї змінної в іншу. Не переносить, а саме копіює. Так само, як копіюють файл з диску на диск. Присвоювання має такий вигляд: ім'я = значення; де ім'я — це ім'я змінної, а значення — це значення, яке записується в змінну. Змінній можна присвоїти конкретне значення, ім'я іншої змінної або навіть вираз, в якому використовуються змінні.


int a;// Створюємо змінну а типу int
                        a = 1;// Записуємо їй значення 1
                        int b; // Створюємо змінну b типу int
                        b = 2; // Записуємо їй значення 2

Важливо!В одному методі не можна створити дві змінні з однаковими іменами. А в різних методах — можна.


Є й інші обмеження щодо імен змінних. Хоча змінним можна надавати будь-які імена, однак в цих іменах не можна використовувати пробіли, символи +, - тощо. Найкраще використовувати в іменах змінних лише латинські літери й цифри.


Зауважте, що в мові Java розрізнюються великі й малі літери. тому int a — це не те саме, що Int a



Компілятор і байт-код


Для комп'ютера в ролі команд виступають числа: кожна команда закодована певним числом (його ще називають машинним кодом).


Писати програму у вигляді чисел дуже складно, тому люди вигадали мови програмування та компілятори. Така мова, з одного боку, зрозуміла людині, з іншого — компілятору. Компілятор — це спеціальна програма, яка перетворює текст програми, написаний мовою програмування, на набір машинних кодів.


Зазвичай програміст пише програму мовою програмування, а потім запускає компілятор, який на основі написаних програмістом файлів із кодом програми робить один файл із машинним кодом — кінцеву (скомпільовану) програму.



Отримана в результаті програма відразу може виконуватись на комп'ютері. Такий підхід має свій мінус — код отриманої програми значно залежить від процесора й операційної системи. Програма, скомпільована для Windows, не буде працювати на телефоні з Android.


Якщо ж ви написали програму для Android, то в операційній системі Windows вона не запуститься!


У Java використовується набагато інноваційніший підхід:



Компілятор Java не компілює всі класи в одну програму з машинних кодів. Натомість він компілює кожен клас окремо, і не в машинні коди, а в спеціальний проміжний код (байт-код). Компіляція в машинний код відбувається під час запуску програми.


А хто ж компілює програму в машинний код під час запуску?


Для цього є спеціальна програма під назвою JVM (Java Virtual Machine) — віртуальна машина Java. Спочатку запускають її, а потім — програму, що складається з байт-коду. А вже JVM перед виконанням потрібної програми компілює її в машинний код.



Коментарі в Java


Два види коментарів


Створюючи програму мовою Java, можна писати не лише команди, а й коментарі до них безпосередньо в коді. Компілятор ігнорує такі коментарі, ніби їх взагалі немає. Під час виконання програми всі коментарі пропускаються.


У коді класу ми написали коментар «Це перший вид коментарів. Він може закоментувати декілька рядків». Початок коментаря позначається парою символів /*, а кінець — */. Коли програма компілюватиметься, компілятор пропустить усе, що міститься між символами /* і */.



public class java_course {
                        public static void main(String[] args) {
                            /*Це перший вид коментарів,
                            Він може закоментувати декілька рядків*/
                        }
                    }

У тексті коментаря ви можете написати що завгодно.


Зазвичай в коді пишуть різні коментарі до його неочевидних або складних для розуміння частин. Трапляються коментарі на десятки рядків: такі зазвичай пишуть перед методами, описуючи нюанси їх роботи.


Є ще один спосіб створити коментар у коді — за допомогою символів //.


public class java_course {
                            public static void main(String[] args) {
                                // Це другий вид коментарів
                                // Він може закоментувати лише один рядок
                            }
                        }


У такому разі коментарем вважається частина коду, що починається з пари символів // і продовжується до кінця рядка, в якому ці символи розташовані. Тобто другої пари символів, які «закривають коментар», немає.
